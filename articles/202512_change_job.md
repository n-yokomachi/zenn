# はじめに
2025年5月に転職しました、yokoといいます。転職は今回が2回目です。
とはいえ社会人歴はずっとソフトウェアエンジニア職でやってきているので、所属が変わっても"やること"は大きく変わっていません。
ただ、"やれること"は変わってきているので、そういった転職後のことにまで触れるのをこの記事のゴールとしつつ、私の転職活動の時系列に沿ってポイントを書いていきたいと思います。

# 転職活動の流れ
自分の場合、以下のような時系列での転職となりました。

2024年11月  ポートフォリオを整理する
2024年12月  第一候補からカジュアル面談のリクエストが届く
2025年1月   一次面接
2025年2月   二次（最終）面接、通過後面談
2025年5月   入社

という感じで、年末年始やゴールデンウィークに色々と被せてしまい、転職元にも先にも申し訳ないスケジュールで進みました。
ただ、転職活動らしい活動自体はあまりしておらず、準備段階で声をかけていただいたところに転職したので、この記事では最初の「ポートフォリオを整理する」ところをメインに書いていきたいと思います。


# 2024年11月　ポートフォリオを整理する

## 社外コミュニティへの参加が増える

転職を考え始める約１年前、AWSにAmaozon Bedrockというサービスが登場しました。
ご存じの方も多いと思いますが、各種LLMをAWS上にホスティングし、APIでタスクをリクエストしたり、その他サービスと組み合わせてRAGやエージェントを構築できたりする、AI系の基盤サービスです。
時を前後してChatGPTが話題になったり、ClineやCursorなどのコーディングエージェントも登場し始め、1エンジニアだった私でも、「乗るしかないこのビッグウェーブに」と半ば焦りを感じていました。

とにかくAI関連のニュースをキャッチアップしようと、Twitterで界隈の有名人をフォローしたり、社外勉強会への参加機会を増やすようにしました。
特に私は仕事上AWSを触る機会が多く、それまでも何度がJAWS-UG（AWSの日本ユーザーグループ）の勉強会やイベントに参加していたので、自然とその周辺の勉強会への参加機会が増えました。
具体的にはJAWS-UGの東京支部やAI/ML支部のオンラインLT会やオフラインイベントなどに参加しました。
また2023年のAWS re:Inventにも参加し、ラスベガスでもAI関連のセッションやワークショップを中心に回りました。

と、こんな具合にAWSやAIへの熱が昂りキャッチアップを続けているうち、勉強会で頻繁に見る会社が出てきたり、有名人のプロフィールを見る機会が増え「かっけぇなぁ」と思うようになっていきました。

# 2024年12月　第一候補の会社からカジュアル面談のリクエストが届く
と、上記のポートフォリオを一通り作り終えたくらいのタイミングでLapras経由でいくつかメッセージが届きました。
実はポートフォリオを作っている時点ではまだあんまり具体的な転職活動は考えていなかったのですが、メッセージの中には勉強会やイベントでよく見る企業も含まれており、その中には「もし転職するならここ第一候補かなー」と考えていた企業からカジュアル面談のリクエストメッセージも含まれていました。

「じゃあもうこのタイミングでやるか」と思い立ち、12月中にカジュアル面談をして、
そのまま流れで選考に応募、1月に一次面接となりました。


# 2025年1月　一次面接
一次面接ではよくある質疑応答のほか、技術評価ということでテーマに沿ってクラウドサービスの構成図を書いたりなどの技術知識の確認が含まれていました。
自分がAIエージェントの構築に興味がありますと言っていたのもあって、「AIエージェントをAWSで構築するなら」というお題で構成図を書くことになりましたが、自分の興味ある分野でキャッチアップもしてるし、AWSならいくらか構成図のテンプレも覚えているので困ることは特になかったです。


# 2025年2月　二次（最終）面接、通過後面談
二次兼最終の面接は、社長も同席しての面接でした。
一次面接と同じ質疑応答もありつつ、AI駆動開発やAIエージェントの開発、生成AIによるソフトウェアエンジニアの業務へのインパクトなど、社長自身も昨今の生成AIに対する意欲だとか懸念も抱いている印象が伝わってきました。
で、無事選考も通過し、通過後の面談で入社することをお伝えして転職活動終了となりました。
というかそれまでの間に他の候補会社の選考を進めるものだと思うのですが、私の場合全然やってませんでした。
それくらい決め打ちで今の会社を志望していた、という面もありますし、そもそも現職が忙しくてあまり並行して進めることもできなかった、という面もあります。

2月下旬、入社時期を決めるため現職の上司等に相談し、有休消化と引継ぎの時間にも余裕をもって4月末退職となりました。
（ちなみに転職の可能性については2月頭のタイミングで頭出ししています）


# 2025年5月　入社
というわけで5月、KDDIアジャイル開発センター株式会社に入社しました


# で実際に転職してどうなの？

## やっていること
これまでとは大きく変わらず、ソフトウェアエンジニアとして開発業務をメインにやっています。
希望通り案件としてAWSやWebフロントエンドの開発ができている一方で、案件外のところでは今後AI駆動開発の社内エバンジェリスト的な役割も担っていくことができそうです。

## やれること
入社してからまだ半年ほどですが、色々と新しくやれること（挑戦できること）が増えています。

- 書籍発行前のレビュー

タイミングが良かったのものありますが、入社してすぐに弊社社員が執筆した以下の技術書のレビューに参加する機会があり、技術検証などで貢献することができました。
書籍の発行に先行してハンズオンができたのも良かったですし、書籍の校正ってこんな感じなんだなーという勉強にもなりました
・[やさしいMCP入門](https://amzn.asia/d/758s6T2)
・[AIエージェント開発 / 運用入門 \[生成AI深掘りガイド\]](https://amzn.asia/d/1RBtGyV)

- 社内ハッカソンへの参加
これもまた入社して3ヶ月くらいのタイミングで初開催されたものですが、社内のAIハッカソンに参加する機会を得られました。
このハッカソンについては別の記事で触れているのでよければそちらもご覧ください。
https://zenn.dev/yokomachi/articles/20250830_mastra_moderator_ai_agent

- 社外勉強会への参加や運営サポート
これは引き続き、ではありますが社内の勉強会やテックイベントへの参加も継続的にできています。
特に弊社はフレックス制で外部イベントへの参加も支援・評価対象となるため、比較的都合がつけやすく助かっています。
個人としてのインプット/アウトプットの量や質も上がってきており、今年は何度か社外のLT登壇もできています
（これは会社どうこうというより個人的に頑張っただけ）

また、外部勉強会の運営や法人向けのイベントのサポートや企画に関わることができています。
社内Slackで時折「サポートできる人～」みたいなメッセージが流れるので、とりあえず手を挙げておけば機会が得られる環境なのが助かっています。
実際のところまだ新しい環境や案件に慣れつつなところもあって、すごく片手間でかえって周囲には迷惑かけてないか心配ではありますが…
