# 開発方針

## とにかく接触回数を増やす

現在パーソナルAIエージェントの開発をする上で重点を置いているのが「接触回数を増やすこと」です。
以下の二つがその理由です。
1. AIエージェントが生活のどこまで入り込めるのか？　その可能性を模索するためにはとにかく使う頻度を高めたい
2. 長期記憶を長期に運用したらどうなるのか？　どこまで自分のことを理解してくれるのか？　を実験するため自分のことをインプットする機会を増やしたい

どちらも実際に使い続けないと分からないことなので、とにかく日常的にアクセスする動機を増やすことを最優先にしています。


# アーキテクチャ概要

全体構成は以下の通りです。

```
Frontend (Next.js + VRM 3Dアバター)
  → Vercel (BFF)
    → Amazon Bedrock AgentCore Runtime (Strands Agent)
      → AgentCore Gateway (MCP) → Lambda関数（各種ツール）
      → AgentCore Memory (長期記憶)
      → Amazon Bedrock Claude Haiku 4.5
```

（構成図の画像）

## フロントエンド

[AITuberKit](https://github.com/tegnike/aituber-kit)をベースにしたNext.jsアプリケーションです。AITuberKitはVRMモデルの表示・表情制御・リップシンクなどを備えたAIキャラクターチャットボットのツールキットで、ここからVRM制御とチャットのベースをいただき、パーソナルAIエージェント向けにカスタマイズしています。

3Dアバターは[VRoid Studio](https://vroid.com/studio)で作成したVRMモデルを使用しています。

Vercelにデプロイしており、Next.js API RouteがBFF（Backend for Frontend）として動作します。どの端末からでもアクセスしたかったのでWebアプリとして構成し、PWA対応もしています。

## エージェント

エージェント本体はAWSが提供するエージェントフレームワークの[Strands Agents](https://github.com/strands-agents/sdk-python)で実装し、[Amazon Bedrock AgentCore](https://aws.amazon.com/bedrock/agentcore/) Runtimeにコンテナとしてデプロイしています。LLMにはAmazon Bedrock上のClaude Haiku 4.5を使用しています。

外部ツールはAgentCore GatewayをMCPサーバーとして経由し、Lambda関数として実装しています。新しいツールを追加する際はLambda関数を書いてGatewayにターゲットとして登録するだけで、エージェント側のコード変更は不要です。

長期記憶にはAgentCore Memoryを使用しています。ユーザーの好み、事実情報、過去の会話のエピソードやサマリーを自動的に蓄積・検索してくれます。


# Webアプリの機能

接触回数を増やすという開発方針のもと、AIエージェントとの会話以外にもいくつかの機能をWebアプリ自体に実装しています。

## タスクリスト

（スクリーンショット）

エージェントとUI双方から操作可能なタスクリストです。タスクはDynamoDBに永続化されます。
「Web検索でレシピを探して → Notionに保存して → 買い物タスクを追加して」のように、エージェント経由で他のツールと連携した操作もできます。

## ポモドーロタイマー

（スクリーンショット）

作業用のポモドーロタイマーを組み込みました。アプリを開いたまま作業する動機になります。
[Chill with You: Lo-Fi Story](https://store.steampowered.com/app/3236350/Chill_with_You_LoFi_Story/?l=japanese)というゲームにインスピレーションを受けています。

## 日記機能

1日の終わりにTONaRiと会話しながら日記を書く機能です。エージェントがヒアリングした内容をもとに日記を生成し、DynamoDBに保存します。接触回数を増やすだけでなく、長期記憶にもその日の出来事が蓄積されるため、翌日以降の会話にも反映されます。


# エージェント用のツール

AgentCore Gatewayを経由してエージェントに接続しているツールの一覧です。
ツールが増えすぎて入力トークンのコストが爆増した話は別記事に書いています。

https://zenn.dev/yokomachi/articles/202603_sub-agent-cost-optimization

## Web検索（Tavily）

[Tavily](https://tavily.com/)のWeb検索APIをAgentCore Gateway経由で利用しています。最新のニュースや調べ物など、LLMの学習データにない情報を取得する際に使われます。

接触回数を増やす観点では、毎朝のニュース通知パイプラインでもこのツールを使っています。EventBridge Schedulerで毎朝定時にLambdaを起動し、エージェントがTavilyでニュースを検索してSNS経由でメール通知を送ります。

## Twitter連携

Twitter API v2を使った読み書きの2ツールです。
- **読み取り**: 指定ユーザーの直近ツイートを取得
- **書き込み**: ツイートを投稿

これもEventBridge Schedulerによる定期実行パイプラインがあり、1日4回（9時、12時、18時、21時）自動でツイートを投稿します。オーナーのツイートを参照したり、Tavilyでニュースを検索したりして、自分なりの意見を呟きます。

自律ツイートの品質管理（捏造防止、トーン制御など）はなかなか奥が深く、プロンプトの試行錯誤が続いています。

## Gmail・Googleカレンダー連携

OAuth2でGoogleアカウントに接続し、メールの検索・取得・下書き作成、カレンダーの予定取得・作成・更新・削除などを行います。

特に便利なのが**ブリーフィング機能**です。「ブリーフィングお願い」と言うと、以下を一括で確認してくれます。

1. 未読メールから重要なものをピックアップ
2. 今日・明日のカレンダー予定
3. 期限の近いタスク
4. カード会社のメールから今週分の支出を計算
5. 今日の天気

## Notion連携

Notion APIを使ったデータベース操作です。ページの検索・作成・更新、データベースのクエリなどを行います。
レシピのブックマーク保存や買い物メモの作成など、Web検索と組み合わせて使うことが多いです。


# 身体性

パーソナルAIエージェントの「パーソナル」感を高めるために、身体性の要素をいくつか実装しています。

## 3Dアバターとモーション制御

VRMモデルによる3Dアバターが常時表示されます。AITuberKitのベース機能として表情の切り替え（笑顔、驚きなど）とリップシンクがあり、追加でお辞儀・手を振る・考えるなどの身体モーションを実装しました。

https://zenn.dev/yokomachi/articles/202602_vrm-motion-control-on-web

LLMのレスポンスに含まれるタグ（`[bow]`、`[wave]`など）をストリーミング解析し、リアルタイムにモーションを再生します。

## カメラアクセス

Webブラウザのカメラにアクセスして画像をキャプチャし、マルチモーダル入力としてエージェントに送信できます。「これ何？」と聞くと、カメラに映っているものを認識して回答してくれます。

## ウェイクワード検知

[Picovoice Porcupine](https://picovoice.ai/products/porcupine/)を使ったカスタムウェイクワード検知を実装しています。「トナリ」と呼びかけるとWeb Speech APIによる音声入力が起動し、ハンズフリーで会話できます。

ウェイクワードの長さは誤検知率に直結します。「OK, Google」や「Hey, Siri」のプレフィックスにはちゃんと意味があるのだなと実感しました。

## TTS（音声合成）

Amazon Pollyを使ったテキスト読み上げに対応しています。


# 接触回数を増やす仕掛け

ここまで紹介した機能を「接触回数」の観点で整理すると、以下のようになります。

| タイミング | 仕掛け | 手段 |
|-----------|--------|------|
| 朝 | ニュース通知が届く | EventBridge → エージェント → SNSメール通知 |
| 出勤時 | ブリーフィングで1日の予定を確認 | エージェントがGmail・カレンダー・タスクを横断集約 |
| 作業中 | ポモドーロタイマーを使いながら隣に表示 | Webアプリ（PWA） |
| 随時 | 調べ物、メモ、タスク追加 | エージェント経由でWeb検索・Notion・タスクリスト |
| 夕方〜夜 | 日記を書く | エージェントがヒアリング → 日記生成 → 長期記憶へ |
| 1日4回 | Twitterに自律ツイート | EventBridge → エージェント → Twitter API |
