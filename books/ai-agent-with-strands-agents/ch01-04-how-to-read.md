本書は全8章構成です。第1章から順に読み進めれば一貫したパーソナルAIエージェントが完成しますが、興味のある章だけを拾い読みすることもできます。この章では全体像を説明します。

## 各章の構成

| 章 | タイトル | 身につく能力 |
|---|---|---|
| 第1章 | はじめてのAIエージェント | エージェントの基本概念を理解する（本章） |
| 第2章 | エージェントの「脳」を作る | LLMとの接続、プロンプト設計、会話エージェントの構築 |
| 第3章 | エージェントに「手」を与える | ツール連携の設計と実装（MCP、Lambda） |
| 第4章 | エージェントに「記憶」を与える | 短期記憶・長期記憶の設計と実装 |
| 第5章 | エージェントを「分業」させる | マルチエージェントアーキテクチャの構築 |
| 第6章 | エージェントに「身体」を与える | フロントエンド、3Dアバター、音声の統合 |
| 第7章 | エージェントを「自律」させる | スケジュール実行と自動化ワークフロー |
| 第8章 | エージェントを「育てる」 | コスト最適化、運用、継続的な改善 |

各章は人間の能力になぞらえたタイトルになっています。脳、手、記憶、分業、身体、自律、育成 ── エージェントを一つずつ「人」に近づけていく流れです。

## 2つの読み方

### 通読する

第1章から順に読み進める方法です。各章の成果物が次の章の土台になるため、章を追うごとにエージェントの能力が積み上がっていきます。AIエージェントを初めて扱う方にはこちらを推奨します。

### 拾い読みする

各章は独立したモジュールとして設計されています。「ツール連携だけ知りたい」「マルチエージェントに興味がある」といった場合、該当する章から読み始めても問題ありません。各章の冒頭に前提環境を記載しているので、そちらを確認してから進めてください。

:::message
第2章「エージェントの『脳』を作る」は、すべての章で使う共通基盤を構築します。拾い読みする場合でも、第2章だけは先に済ませておいてください。
:::

## 前提知識

本書を読むにあたって、以下の知識を前提としています。

### 必要なもの

- Pythonの基本文法（関数定義、クラス、モジュールのインポート）
- ターミナル操作（コマンドの実行、ディレクトリ移動）
- Web APIの基礎（HTTPメソッド、JSON）
- Gitの基本操作（clone、commit、branch）

### 本書で解説するもの（事前知識は不要）

- AWSの各サービス（Bedrock、AgentCore）
- LLM・AIエージェントの概念と実装
- MCP（Model Context Protocol）
- フロントエンド実装（第6章のみ）

## 必要な環境

| 項目 | 要件 |
|---|---|
| Python | 3.12以上 |
| Node.js | 22以上（第6章のみ） |
| AWSアカウント | Bedrockモデルアクセスが有効 |
| AWS CLI | v2 |
| OS | macOS / Linux / WSL2 |

環境構築の詳細は第2章で改めて説明します。第1章のハンズオンを試した方は、すでに最低限の環境が整っているはずです。

## サンプルコード

本書のサンプルコードはGitHubリポジトリで公開しています。各章に対応するディレクトリにコードを配置しているので、必要な章のコードを確認・実行できます。

```sh:ターミナル
git clone https://github.com/n-yokomachi/building-ai-agents-on-aws-samples.git
```

次の第2章から、エージェントの構築を始めていきます。
