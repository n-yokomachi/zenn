前章では10行のコードでAIエージェントを動かしました。「天気を調べて」と頼むだけで、エージェントが自らWebにアクセスし、情報を取得して回答を返してくれました。このプログラムは、普段使っているチャットボットとどこが違うのでしょうか。そして、なぜこんな動きが可能なのか。このコードの中身を分解しながら、AIエージェントの正体を明らかにしていきます。

## チャットボットとの違い

ChatGPTやClaude.aiのようなチャットボットを使ったことがある人は多いでしょう。AIエージェントとチャットボットは、どちらもLLM（大規模言語モデル）を使っている点では同じです。しかし、できることには明確な差があります。

| | チャットボット | AIエージェント |
|---|---|---|
| できること | 学習済み知識をもとに回答する | 外部の情報を取得し、行動した上で回答する |
| 情報の範囲 | 学習データに含まれる知識のみ | ツールを通じてリアルタイムの情報にもアクセスできる |
| 主体性 | ユーザーの質問に受動的に応答する | 目的を達成するために自ら手段を選んで行動する |

チャットボットに「東京の今の天気は？」と聞くと、「最新の天気情報は取得できません」と返されるか、学習データに含まれる過去の傾向をもとにした推測が返ってきます。一方、前章のエージェントは実際にWebサイトへアクセスし、リアルタイムの天気情報を取得して回答しました。

この違いを一言でまとめると、チャットボットは「知っていることを答える存在」、AIエージェントは「考えて行動する存在」です。

## エージェントの3要素

では、エージェントはどのようにして「考えて行動する」を実現しているのか。前章のコードを再掲します。

```python:agent.py
from strands import Agent
from strands_tools import http_request

agent = Agent(
    model="us.anthropic.claude-3-5-haiku-20241022-v1:0",
    tools=[http_request],
    system_prompt="あなたは親切なアシスタントです。日本語で回答してください。",
)

response = agent("東京の今の天気を調べて教えてください")
print(response)
```

このコードには、AIエージェントを構成する3つの要素が詰まっています。順に見ていきましょう。

### 脳 ── LLM

```python
model="us.anthropic.claude-3-5-haiku-20241022-v1:0"
```

`model=` で指定しているのは、エージェントの頭脳となるLLMです。ユーザーからのリクエストを理解し、何をすべきか考え、最終的な回答を組み立てる。人間でいえば脳にあたる部分です。

LLMは自然言語を理解して生成する能力を持っていますが、それだけでは外の世界の情報を得ることも、何かを操作することもできません。しかし脳だけでは外の世界に手が届きません。そこで次の要素が必要になります。

### 手 ── ツール

```python
tools=[http_request]
```

`tools=` で渡しているのは、エージェントが外の世界に触れるための道具です。前章では `http_request` という、Webページを取得するツールを1つだけ渡しました。このツールがあることで、エージェントはインターネット上の情報にアクセスできるようになります。

ツールは必要に応じて追加できます。ファイルの読み書き、データベースへの問い合わせ、外部APIの呼び出し ── ツールを増やすほど、エージェントの手は広がっていく。第2章以降で、さまざまなツールを実際に追加していきます。

### 判断力 ── エージェントループ

3つめの要素は、コードの中に明示的には現れません。しかし、エージェントをエージェントたらしめる最も重要な仕組みです。

`agent("東京の今の天気を調べて教えてください")` を呼び出したとき、内部では次のようなループが回っています。

1. ユーザーの入力をLLMに渡す
2. LLMが「何をすべきか」を判断する
3. ツールの実行が必要と判断された場合、ツールを呼び出す
4. ツールの実行結果をLLMに戻す
5. LLMが「十分な情報が揃ったか」を判断する
6. 十分であれば回答を生成し、不十分であればステップ2に戻る

これがエージェントループと呼ばれる仕組みです。前章の天気の例では、LLMがまず「天気情報を得るにはWebにアクセスする必要がある」と判断し、`http_request` ツールを使って天気サイトにアクセスし、取得した結果から必要な情報を抜き出して回答を組み立てました。

重要なのは、この一連の流れを開発者がステップごとに指示していない点です。「天気を調べて」という目的だけを与え、どのURLにアクセスし、どの情報を抽出するかはエージェント自身が判断している。この自律的な判断のループこそが、チャットボットとの決定的な違いです。

## system_promptの役割

ここまでの3要素（LLM・ツール・エージェントループ）に加えて、もう1つ注目しておきたいパラメータがあります。

```python
system_prompt="あなたは親切なアシスタントです。日本語で回答してください。",
```

`system_prompt` は、エージェントの人格や行動指針を定義するテキストです。現時点では「親切に、日本語で」という簡単な指示だけですが、この設定がエージェントの振る舞いを大きく左右します。

たとえば「回答は必ず箇条書きにしてください」と書けば出力形式が変わり、「不確かな情報は推測せず、わからないと答えてください」と書けば慎重な振る舞いになる。system_promptはエージェントに与える行動規範であり、エージェントの個性を決める重要なパラメータです。第2章でこの設計を深掘りしていきます。

## まとめ

AIエージェントとは、LLM（脳）、ツール（手）、エージェントループ（判断力）の3要素で構成されるシステムです。LLMが考え、ツールで行動し、結果を見て次の判断を下す ── この自律的なループによって、単なるチャットボットを超えた「考えて行動する存在」が実現されています。

本書では、この3要素をひとつずつ拡張しながら、実用的なAIエージェントを組み上げていきます。次章では、なぜ今このタイミングでAIエージェントが注目されているのか、その背景を整理します。
