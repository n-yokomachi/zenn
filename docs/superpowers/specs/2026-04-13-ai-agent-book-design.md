# AWSで作るパーソナルAIエージェント入門 ― ハンズオンで学ぶ Strands Agents & Amazon Bedrock AgentCore

設計ドキュメント | 2026-04-13

## 概要

TONaRiプロジェクトをベースに、AIエージェントの構築方法を解説する技術書。読者が自分なりのパーソナルAIエージェントの基盤を構築できるようになり、かつエージェントの設計思想・アーキテクチャを理解して必要な部分を選んで実装できることを目指す。

将来的には技術書典等での物理本販売も視野に入れるが、まずはZenn Bookとして執筆・公開する。

## 想定読者

- プログラミング経験はあるがAI/エージェント開発は初めての開発者
- Python、AWS等の基礎的な説明は不要
- AWSを触ったことがない読者も想定し、環境構築のハンズオンは含める
- プログラミング言語の基礎など初歩的な内容は含めない

## 技術スタック

メインスタック（TONaRiと同一）:
- Strands Agents SDK（Python）
- Amazon Bedrock AgentCore
- Claude Haiku 4.5（LLM）
- AgentCore Gateway / MCP
- AWS Lambda（ツール実装）
- AgentCore Memory（記憶）
- Next.js + VRM（フロントエンド）

コラムで紹介する代替技術:
- Google Vertex AI / ADK
- LangGraph / LangChain
- OpenAI Agents SDK
- OpenRouter（モデル切り替え）

## 構成アプローチ

**能力獲得型（モジュール構成）**

- 各章がエージェントの「能力」を1つずつ扱う
- 各章は独立して読める（モジュール型）
- 通読すると複数サービス連携が可能な完成されたAIエージェントが出来上がる
- 第2章のみ全章共通の基盤として「必須章」と位置づける

### モジュール独立性の実現方法

- 各章の冒頭に「この章で必要な前提環境」を明記
- 各章で使うコードはGitHubリポジトリのブランチまたはタグで提供し、途中から始められるようにする
- 第2章だけは全章の共通基盤なので、他章から参照される必須章

### コラム・Tips

- コラム: 各章に1〜2個。代替技術の紹介、設計判断の裏話
- Tips: 実装上のハマりどころ、パフォーマンス改善、デバッグ手法

## 第1章「はじめてのAIエージェント」詳細設計

### 章の目的

読者にまず最小限のAIエージェントを動かす体験をさせ、その後「今動かしたものは何だったのか」を解説することで、AIエージェントの概念を体感的に理解させる。

### 構成

#### 1.1 まずは動かしてみよう（ハンズオン）

- Strands Agents SDKのインストール（pip install）
- Bedrock有効化の最低限の手順
- 10行程度のコードで「天気を調べて」と聞いたらツールを使って答えるエージェントを動かす
- 「これがAIエージェントです」という着地

**ポイント**: 最短距離で動かす。細かい説明は後回しにし、「写経して動いた！」という体験を優先する。

#### 1.2 今あなたが動かしたものは何か

- チャットボットとエージェントの違い
- エージェントの定義：「LLM + ツール + 判断力」
- 1.1のコードを振り返りながら、どこが「脳」でどこが「手」かを図解

**ポイント**: 1.1で動かしたコードを分解して「エージェントの構造」を理解させる。

#### 1.3 なぜ今、AIエージェントなのか

- LLMの進化とエージェントの関係（簡潔に）
- 個人開発者でもエージェントが作れる時代
- TONaRiの紹介：本書で作り上げるエージェントの全体像

**ポイント**: 長い歴史解説は不要。「今なぜ」に絞る。

#### ※ 1.4 本書の歩き方 → 「はじめに」に統合済み

本書の構成、読み方ガイド、前提知識、開発環境、サンプルコードリポジトリの案内は「はじめに」（ch00-introduction）に集約した。

#### [コラム] エージェントフレームワークの選択肢

- Strands Agents / LangGraph / OpenAI Agents SDK / Google ADK 等を簡単に比較
- なぜ本書ではStrands Agentsを選んだのか
